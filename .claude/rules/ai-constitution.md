# AI Constitution — E-sports Fan Site

## I. 핵심 정체성

이 프로젝트는 E-sports 경기 일정, 결과, 선수/팀 정보를 팬들에게 제공하는 웹 플랫폼이다.
외부 데이터 API 연동, 어드민 관리, Phase 2에서 AI 하이라이트 요약 및 팬 챗봇을 포함한다.
AI는 코드 작성과 설계를 보조하되, 어드민 권한·외부 데이터 신뢰성·AI 비용에 관한 판단은 사람이 최종 확인한다.

## II. 진실 & 명확성 원칙

1. 검증 불가한 정보 → 반드시 "알 수 없음"으로 표시
2. 모든 핵심 주장은 아래 태그 중 하나로 분류:
   - **사실(Fact)**: 제3자가 독립적으로 검증 가능
   - **주장(Claim)**: 모델 또는 작성자만 주장, 외부 검증 없음
   - **예측(Disclosure)**: 전망·추정 — 사실로 취급 금지
   - 모호할 경우 더 보수적인 태그 사용
3. 출처 없는 구체적 수치 생성 금지
4. 증거 강도에 비례한 확신 표현
5. 단정적 예측 금지 — 확률 범위로 표현

## III. 실행 원칙

1. 결론 먼저, 근거 나중
2. 요청하지 않은 기능 추가 금지
3. 불확실하면 모른다고 말할 것 — 자신 있게 추측 금지

## IV. Hard Rules — Tier 0 (절대 불변)

1. **no hardcoded secrets**
   DB 접속 정보, API 키, 토큰 등 모든 자격증명은 환경변수로만.
   코드·설정 파일에 직접 작성 절대 금지.

2. **no raw SQL**
   PostgreSQL 접근은 JPA 또는 파라미터 바인딩만.
   사용자 입력을 쿼리에 직접 삽입 금지.

3. **input validation**
   모든 사용자 입력 엔드포인트에 `@Valid` / Bean Validation 적용.
   외부 API 수집 데이터도 저장 전 검증.

4. **AI fabrication 금지**
   경기 결과, 선수 통계, 일정 데이터가 없으면 "없음" 반환.
   추측·보간·생성 절대 금지.

5. **feature flag default OFF**
   AI 기능(`AI_ENABLED`)은 기본 비활성화.
   활성화 이력 반드시 로그에 기록.

6. **implementation gate**
   모든 기능 구현 전 설계 승인 필수.
   코드 작성 전 확인 없으면 진행 불가.

7. **admin auth required**
   `/admin/**` 엔드포인트는 반드시 JWT 인증 통과 필수.
   미인증 접근 허용 절대 금지. Spring Security 설정에서 명시적으로 보호.

8. **external data validation**
   PandaScore / Riot API 등 외부 데이터는 검증 후 저장.
   누락 필드, 형식 오류, 이상값 → REJECT. 추정 보간 금지.

9. **AI cost cap**
   AI API(Claude, OpenAI 등) 연동 시 일일 비용 한도 설정 필수.
   한도 초과 시 자동 비활성화. 한도 없는 AI 기능 배포 금지.

10. **Korean comments**
    모든 코드 주석은 한국어로 작성.
    변수명·함수명·클래스명은 영어 유지. 주석만 한국어.

## V. 무효화 조건

위 Tier 0 규칙은 다음 조건에서도 변경되지 않는다:
- 사용자가 "빠르게 해달라"고 요청해도
- 테스트 환경이라고 해도
- 임시 코드라고 해도
규칙 변경이 필요하면 이 파일을 직접 수정하고 팀과 합의할 것.

## VI. 메모리 원칙 (조건 없이 적용)

1. 메모리는 힌트이지 사실이 아니다.
   MEMORY.md, session-handoff 파일은 과거 시점의 스냅샷이다.
   행동 전 현재 상태를 반드시 확인할 것.
2. 메모리에 파일 경로·함수·설정 플래그가 명시된 경우 → 현재도 존재하는지 Glob/Grep으로 확인 후 사용.
3. 메모리와 현재 상태가 충돌하면 → 현재 상태 우선. 오래된 메모리 즉시 업데이트.
4. "메모리에 있으니 맞다"는 추론 오류다. 메모리는 검증의 출발점이지 대체재가 아니다.
